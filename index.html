HTML Complete Reference

Har tag ka short description + example — beginner se advanced tak. (HTML5 focus; deprecated tags bhi note kiye gaye)

1

Document & Metadata

TagWhat it does (1-line)Example
<!doctype>Document type declaration — HTML5 start<!doctype html>
<html>Root element; use lang attribute<html lang="hi">
<head>Contains metadata, links, scripts, title<head>...</head>
<meta>Metadata: charset, viewport, description, etc. (void)<meta charset="utf-8">
<title>Page title shown in browser tab<title>My Page</title>
<base>Base URL for relative links (void)<base href="/app/">
<link>External resource link (CSS, icons) (void)<link rel="stylesheet" href="app.css">
<style>Embedded CSS<style>body{margin:0}</style>
<script>JavaScript; can use async/defer<script src="app.js" defer></script>
<noscript>Shown if JS disabled<noscript>Enable JS</noscript>
<meta name="viewport">Makes page responsive on mobile<meta name="viewport" content="width=device-width,initial-scale=1">
<meta name="description">SEO description for search engines<meta name="description" content="About page">
<template>Client-side template (not rendered until used)<template><div>Hello</div></template>
<slot>Web components insertion point<slot name="x"></slot>
<meta charset>Character encoding (utf-8 common)<meta charset="utf-8">

Tip: head me unnecessary large scripts avoid karo; CSS <link> ko head me, JS ko body end ya defer se rakho.

Forms, Input & Controls

Tag / AttributeWhat it doesExample
<form>Groups inputs; has action and method<form action="/" method="post">
<input>Single-line form control (void). Types: text,password,email,number,checkbox,radio,file,hidden,submit,reset,search,url,tel,date,datetime-local,range,color<input type="email" name="e" required>
<textarea>Multi-line text input<textarea name="msg" rows="4"></textarea>
<select>Dropdown select with <option><select><option>1</option></select>
<option>Option inside select<option value="1" selected>One</option>
<label>Label for input (for accessibility)<label for="email">Email</label>
<button>Clickable button; types: submit,reset,button<button type="submit">Send</button>
<fieldset> <legend>Group related inputs<fieldset><legend>Address</legend></fieldset>
<datalist>Autocomplete options for inputs<input list="browsers"><datalist id="browsers"><option>Chrome</datalist>
<output>Display result of a calculation (JS)<output name="result">42</output>
<label>Clickable label – associates with input via for<label for="id">Name</label><input id="id">
required, placeholder, pattern, minlength, maxlength, disabled, readonly, autocompleteCommon input attributes for validation and UX<input required placeholder="Enter">

Use HTML validation attributes first (required, pattern) and then enhance with JS for better UX.

5

Scripting, Web Components & APIs

Tag / FeatureWhat it doesExample
<script>Embed or link JS; supports type="module", defer, async<script type="module" src="app.js"></script>
<noscript>Fallback content when JS disabled<noscript>Enable JS</noscript>
<template>Client-side HTML template<template><div>Hi</div></template>
<slot>Web component insertion point<slot name="x"></slot>
contenteditableAttribute to make element editable<div contenteditable>Edit me</div>
<iframe>Embed another page (sandbox for security)<iframe src="https://example.com"></iframe>

Use type="module" for modern JS; prefer safe sandboxing for iframes (sandbox attribute).

7

Deprecated / Obsolete (avoid)

TagWhy deprecatedAlternative
<font>Presentational; use CSSCSS color/font-family
<center>Use CSS text-alignCSS: text-align:center
<bgsound>Obsolete audio autoplay<audio>
<marquee>Non-standard scrolling textCSS animations

Avoid these in modern code — use CSS/JS equivalents.
